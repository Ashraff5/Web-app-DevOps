DevOps Project Overview

Project Description

This project demonstrates my proficiency in deploying a modern web application with a DevOps workflow. I utilized a range of tools and technologies to containerize, deploy, and manage the application in a cloud-native environment.

Key Steps

  • Containerization: Created a Dockerfile to containerize the web application, ensuring portability and consistency across environments.
  • Docker Registry: Built the Docker image and pushed it to a secure container registry for centralized access.
  • Kubernetes Manifest Repo: Organized Kubernetes manifests under a k8s/manifests repository to manage configurations.
  • Kubernetes Cluster: Set up a Kubernetes cluster using Amazon EKS, ensuring scalability and reliability for the application.
  • Ingress Controller: Configured both the Ingress resource and Ingress controller to manage application routing and load balancing.
  • Helm Charts: Utilized Helm to package Kubernetes configurations, simplifying the deployment process.
  • Argo CD: Deployed the application into the Kubernetes cluster using Argo CD for GitOps-based continuous delivery.
  • CI/CD Pipeline: Implemented a CI/CD pipeline using GitHub Actions and workflows to automate code integration, testing, and deployment.

Technologies Used

  • Containerization: Docker
  • Orchestration: Kubernetes (EKS)
  • Configuration Management: Helm
  • Continuous Delivery: Argo CD
  • CI/CD Automation: GitHub Actions and GitHub Workflows
  • Cloud Provider: Amazon Web Services (AWS)
